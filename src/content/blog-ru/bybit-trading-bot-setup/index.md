Автоматизация торговли на Bybit позволяет торговать 24/7 без постоянного мониторинга рынка. Эта инструкция — для тех, кто подключает к бирже **внешние** торговые платформы или ботов (сторонние сервисы алготрейдинга): им нужны API-ключи Bybit. Встроенные инструменты на самой бирже Bybit работают без API. Как распределять капитал между несколькими ботами и снижать рисковую нагрузку — в серии [Диверсификация и настройка торговой системы](/ru/library/diversification-trading-system-part-1/).

Можно работать встроенными биржевыми ботами Bybit, а можно подключить ботов других платформ автоматизации торговли — например, [Veles](https://veles.finance/invite/washmallay) (реферальная ссылка: регистрация по ней даёт преимущества и нам, и вам). Если решите подключать внешнюю платформу и нужна помощь — напишите нам; контакты на странице [О проекте](/ru/about/).

## Что нужно для подключения внешнего бота к Bybit

Перед настройкой убедитесь, что у вас есть:

- **Аккаунт на Bybit** — зарегистрированный и верифицированный
- **Средства на счете** — для торговли и оплаты комиссий
- **API ключи** — для подключения внешней платформы или бота к бирже
- **Торговая стратегия** — понимание того, как должен работать бот

## Шаг 1: Регистрация и верификация на Bybit

Если у вас еще нет аккаунта на Bybit, необходимо зарегистрироваться:

1. Откройте [официальный сайт Bybit](https://www.bybit.com/ru-RU/invite?ref=PWMD24) (при регистрации по ссылке — партнёрская поддержка проекта)
2. Нажмите кнопку "Регистрация" в правом верхнем углу
3. Введите email или номер телефона
4. Подтвердите регистрацию кодом из письма или SMS
5. Примите условия пользовательского соглашения

После регистрации рекомендуется пройти верификацию (KYC) для увеличения лимитов на вывод средств и доступа к дополнительным функциям.

## Шаг 2: Пополнение счета

Перед началом торговли нужно пополнить счет на Bybit:

1. Войдите в личный кабинет
2. Перейдите в раздел "Активы" → "Пополнить"
3. Выберите криптовалюту (рекомендуется USDT или USDC)
4. Выберите метод пополнения (банковская карта, P2P, криптовалютный перевод)
5. Следуйте инструкциям для пополнения

**Важно:** После пополнения средства поступают на Funding Account. Для торговли их нужно перевести в Unified Trading Account:

1. Откройте "Активы" → "Перевод"
2. Выберите источник: Funding Account
3. Выберите получателя: Unified Trading Account
4. Укажите сумму и подтвердите перевод

## Шаг 3: Создание API ключа на Bybit

API ключ нужен, чтобы внешняя платформа (бот) могла выставлять и отменять ордера от вашего имени — без доступа к выводу средств. Пошаговая инструкция:

### 3.1. Создание API ключа

1. Войдите в личный кабинет Bybit
2. Нажмите на иконку профиля в правом верхнем углу
3. Выберите "API" в меню
4. Нажмите "Создать новый ключ"
5. Введите название ключа (например, "Trading Bot")
6. Выберите тип API ключа:
   - **Read Only** — только чтение (для аналитики)
   - **Trade** — торговля (для ботов)
7. Пройдите двухфакторную аутентификацию (2FA)

### 3.2. Настройка разрешений API

Для работы внешней платформы (бота) необходимо включить следующие разрешения:

- ✅ **Enable Spot & Margin Trading** — для спотовой торговли
- ✅ **Enable Futures** — для торговли фьючерсами (если используете)
- ❌ **Enable Withdrawals** — **рекомендуется отключить** для безопасности

**Важно:** Разрешение на вывод средств (Withdrawals) для API ключей лучше не включать — это защищает средства при компрометации ключа.

### 3.3. Ограничение доступа по IP (рекомендуется)

Для дополнительной безопасности можно ограничить доступ API ключа по IP-адресам:

1. В настройках API ключа выберите "IP Access Restrictions"
2. Выберите "Restrict access to trusted IPs only"
3. Добавьте IP-адреса серверов, с которых будет работать бот

Если вы не знаете IP-адреса сервера бота, можно оставить этот параметр без ограничений, но это менее безопасно.

## Шаг 4: Подключение API ключа к внешней платформе

После создания API ключа на Bybit его нужно ввести во внешней платформе (сервисе алготрейдинга), которую вы используете:

1. Откройте панель управления вашей платформой
2. Перейдите в раздел «API ключи» или «Подключение биржи»
3. Выберите биржу "Bybit"
4. Введите:
   - **API Key** — публичный ключ из Bybit
   - **API Secret** — секретный ключ (показывается только один раз при создании)
5. Сохраните настройки

**Важно:** Секретный ключ (API Secret) показывается только один раз при создании. Если вы его не сохранили, нужно создать новый API ключ.

## Шаг 5: Настройка торговой стратегии

После подключения API ключа можно настроить торговую стратегию. Рассмотрим основные параметры:

### Выбор торговой пары

Рекомендуется выбирать ликвидные пары с хорошим объемом торгов:

- **BTC/USDT** — самая ликвидная пара
- **ETH/USDT** — вторая по популярности
- **Топ-20 монет** — надежные активы с хорошей ликвидностью

Избегайте малоизвестных монет с низким объемом торгов — они могут быть нестабильны.

### Настройка параметров стратегии

В зависимости от типа стратегии (Grid, DCA, Scalping и т.д.) настройте:

- **Размер позиции** — сколько средств выделить на бота
- **Шаг между ордерами** — расстояние между ордерами в сетке
- **Количество ордеров** — сколько ордеров разместить
- **Стоп-лосс и тейк-профит** — уровни для защиты капитала

Подробнее о риск-менеджменте читайте в статье про [риск-менеджмент в криптотрейдинге с ботами](/ru/library/risk-management-crypto-trading-bots).

### Использование кредитного плеча

Если используете фьючерсную торговлю:

- **Для начинающих:** не более 5–10x
- **Для опытных:** до 20x (с осторожностью)
- **Избегайте:** плечо выше 20x — высокий риск ликвидации

Рекомендуется использовать **изолированную маржу** (Isolated Margin) — это ограничивает риск только средствами, выделенными на позицию.

## Шаг 6: Запуск и мониторинг бота

После настройки всех параметров можно запустить бота:

1. Проверьте все настройки еще раз
2. Убедитесь, что на счете достаточно средств
3. Запустите бота
4. Начните мониторинг результатов

### Что отслеживать

- **Производительность бота** — прибыль/убыток
- **Количество открытых позиций**
- **Просадка (drawdown)** — максимальное падение от пика
- **Исполнение ордеров** — корректность работы

### Когда остановить бота

Остановите бота, если:

- Рынок резко изменился (не подходит для стратегии)
- Достигнут лимит убытков
- Нужно скорректировать параметры
- Обнаружены технические проблемы

## Типичные ошибки при настройке

### 1. Включение разрешения на вывод средств

**Ошибка:** Включение "Enable Withdrawals" в настройках API ключа.

**Последствия:** В случае компрометации ключа злоумышленники могут вывести все средства.

**Решение:** Не включайте это разрешение для ключей, которые использует внешняя платформа.

### 2. Использование слишком высокого плеча

**Ошибка:** Установка кредитного плеча 50x, 100x и выше.

**Последствия:** Высокий риск ликвидации позиции при небольшом движении цены.

**Решение:** Используйте умеренное плечо (5–10x для начинающих).

### 3. Недостаточный объем торговли

**Ошибка:** Выбор торговой пары с низким объемом (< 500 BTC за 24 часа).

**Последствия:** Проблемы с исполнением ордеров, проскальзывание.

**Решение:** Рассматривайте пары из топ-200 с высоким объемом торгов.

### 4. Игнорирование стоп-лоссов

**Ошибка:** Торговля без установки стоп-лоссов.

**Последствия:** Одна неудачная сделка может уничтожить весь депозит.

**Решение:** На начальных этапах рекомендуется всегда устанавливать стоп-лоссы и тейк-профиты.

## Безопасность API ключей

Для защиты средств рекомендуется придерживаться таких правил:

- ✅ Использовать двухфакторную аутентификацию (2FA)
- ✅ Ограничивать разрешения API только необходимыми функциями
- ✅ Регулярно проверять активность API ключей
- ✅ Не хранить секретные ключи в открытом виде
- ✅ Использовать отдельные API ключи для разных платформ или сервисов
- ✅ Отзывать неиспользуемые ключи

## FAQ

**1. Безопасно ли использовать API ключи для подключения внешних платформ?**  
Да, если правильно настроить разрешения: отключить вывод средств и использовать 2FA.

**2. Можно ли использовать один API ключ для нескольких платформ или сервисов?**  
Технически можно, но рекомендуется использовать отдельные ключи для каждой платформы — так проще контролировать доступ и при утечке отозвать только один ключ.

**3. Что делать, если забыл секретный ключ API?**  
Нужно создать новый API ключ. Старый ключ можно отозвать в настройках.

**4. Какая минимальная сумма нужна для торговли через внешнюю платформу на Bybit?**  
Минимальная сумма зависит от стратегии и торговой пары. Обычно достаточно $50–100 для начала.

**5. Можно ли торговать на Bybit без верификации?**  
Да, но с ограничениями. Для полного доступа к функциям биржи рекомендуется пройти верификацию.
