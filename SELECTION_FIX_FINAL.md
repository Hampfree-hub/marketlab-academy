# ✅ ИСПРАВЛЕНИЕ ВЫДЕЛЕНИЯ ТЕКСТА — ФИНАЛЬНОЕ

**Дата:** 2026-01-10  
**Проблема:** При выделении текста на двух строчках всё сливается в один сплошной блок  
**Причина:** `line-height` слишком маленький (1.2-1.4), строки слипаются при выделении  
**Решение:** Увеличен `line-height` для всех заголовков (1.5-1.7)  
**Статус:** ✅ **ИСПРАВЛЕНО**

---

## ❌ ПРОБЛЕМА

При выделении многострочного заголовка:
- Строки сливаются в один сплошной блок выделения
- Нет видимого пространства между строками
- Выглядит ущербно и непрофессионально

**Причина:** `line-height` слишком маленький:
- Базовый для всех h1-h6: `1.4`
- h1: `1.2` (слишком маленький!)
- h1.brutal: `1.1` (очень маленький!)
- h1.long: `1.3` (маленький)

---

## ✅ РЕШЕНИЕ

**Увеличен `line-height` для всех заголовков:**

1. **Базовый для всех h1-h6:**
   - Было: `line-height: 1.4;`
   - Стало: `line-height: 1.6;` ✅

2. **h1 (нейтральный):**
   - Было: `line-height: 1.2;`
   - Стало: `line-height: 1.6;` ✅

3. **h1.brutal (короткие заголовки):**
   - Было: `line-height: 1.1;`
   - Стало: `line-height: 1.5;` ✅

4. **h1.long (длинные заголовки):**
   - Было: `line-height: 1.3;`
   - Стало: `line-height: 1.7;` ✅

**Результат:**
- При выделении многострочного текста между строками теперь видимое пространство
- Выделение выглядит чисто и профессионально
- Каждая строка выделяется отдельно, не сливаясь в один блок

---

## ✅ УНИФИКАЦИЯ

**Все значения `line-height` теперь консистентны:**
- Базовый: `1.6` (для всех h1-h6)
- h1: `1.6` (нейтральный)
- h1.brutal: `1.5` (компактный, но достаточный)
- h1.long: `1.7` (больше пространства для длинных заголовков)

**Преимущества:**
- ✅ Единый стиль выделения по всему сайту
- ✅ Видимое пространство между строками при выделении
- ✅ Профессиональный внешний вид
