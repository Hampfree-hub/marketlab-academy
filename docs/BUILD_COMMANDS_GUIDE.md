# 🚀 Сборка проекта: Инструкция

## Команды сборки

### `npm run build` — Быстрая сборка (разработка)

```bash
npm run build
```

**Что делает:**
- Собирает Astro сайт в папку `dist/`
- **Не генерирует** OG изображения
- Время: ~1-2 минуты

**Когда использовать:**
- ✅ Локальная разработка
- ✅ Быстрая проверка изменений
- ✅ Тестирование вёрстки

---

### `npm run build:full` — Полная сборка (продакшен) ⭐

```bash
npm run build:full
```

**Что делает:**
1. `npm run build` — сборка Astro сайта
2. `npm run generate-og` — генерация OG изображений для всех статей

**Результат:**
- Сайт в `dist/` с готовыми OG изображениями
- Готово к деплою на GitHub Pages
- Время: ~5-10 минут (зависит от количества статей)

**Когда использовать:**
- ✅ Деплой на продакшен
- ✅ Подготовка к публикации новых статей
- ✅ Обновление OG изображений после изменений

---

### `npm run generate-og` — Только OG изображения

```bash
npm run generate-og
```

**Что делает:**
- Генерирует OG изображения для всех статей в `dist/og/`
- Время: ~3-7 минут

**Когда использовать:**
- ✅ Если сайт уже собран, но нужны OG изображения
- ✅ Перегенерация после изменения шаблона
- ✅ Тестирование генерации

---

## 🔄 Рабочий процесс

### Локальная разработка
```bash
npm run dev          # Dev-сервер с hot-reload
```

### Перед коммитом
```bash
npm run build        # Быстрая проверка сборки
```

### Перед деплоем
```bash
npm run build:full   # Полная сборка с OG
npm run preview      # Проверка продакшен-версии локально
```

### Деплой
```bash
git add .
git commit -m "Add new article: ..."
git push origin main
```

GitHub Actions автоматически:
1. Запустит `npm run build`
2. Сгенерирует OG изображения
3. Задеплоит на GitHub Pages

## ⚙️ Как это работает

### `build:full` под капотом

```json
{
  "scripts": {
    "build:full": "npx astro build && npm run generate-og"
  }
}
```

Это **цепочка команд**:
1. Сначала выполняется `npx astro build`
2. Если успешно → выполняется `npm run generate-og`
3. Если Astro сборка упала → `generate-og` не запустится

**Почему так:**
- Безопасность: не генерировать OG для битой сборки
- Скорость: быстрая локальная сборка без OG
- Гибкость: можно запустить OG отдельно

## 🛠️ Troubleshooting

### Сборка упала с ошибкой

```bash
# Очистить кэш
rm -rf node_modules/.vite  # Linux/Mac
rmdir /s /q node_modules\.vite  # Windows

# Переустановить зависимости
npm ci

# Попробовать снова
npm run build:full
```

### OG изображения не генерируются

```bash
# Проверить синтаксис
node --check scripts/generate-og-images.mjs

# Запустить отдельно
npm run generate-og

# Проверить наличие Puppeteer
npm list puppeteer
```

### Долгая генерация

Это нормально для 170+ статей. Для ускорения:
- GitHub Actions кэширует `node_modules`
- Используется системный Chromium (не качается каждый раз)
