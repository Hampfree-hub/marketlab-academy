# Гайд по улучшению статей

> Версия: 1.0 от 2026-03-01
> Цель: сделать статьи полезными, интересными и для новичков, и для профи

---

## 1. Принципы

**Главный принцип:** Статья должна быть полезной сама по себе. Реферальные ссылки и реклама — потом, как следствие, а не основа.

**Лучшие практики:**
- Топовые блоги (CoinDesk, Binance Academy, Investopedia) не используют навязчивые рефералки
- Контент должен быть экспертным и понятным одновременно
- Визуалы — графики и схемы, не стоковые фото
- Структура: H2 → H2 → примеры → выводы

---

## 2. Структура статьи

### Обязательные элементы

| Элемент | Описание | Обязательно |
|---------|----------|-------------|
| **Frontmatter** | title, description, pubDate, category | ✅ |
| **Введение** | 1-2 абзаца, что будет в статье | ✅ |
| **Основной контент** | H2 + объяснение + примеры | ✅ |
| **Итог (Summary)** | Краткое резюме ключевых мыслей | ✅ |
| **FAQ** | 3-6 вопросов по теме | ✅ |

### Что НЕ должно быть

| Элемент | Почему плохо |
|---------|--------------|
| **Заключение** с реферальными ссылками | Навязчиво, как СТА |
| **Связанные материалы** (большой шаблонный блок) | Дублирует навигацию, реклама |
| **Повторяющиеся блоки** Summary + Conclusion | Дублирование |
| **Много рефералок в тексте** | 1-2 максимум, без "register now!" |

### Перекрёстные ссылки (оставить)

Вместо шаблонного блока "Связанные материалы" — **аккуратные ссылки внутри текста**:
- 2-4 ссылки на смежные статьи
- В контексте: "Подробнее о [RSI](/ru/library/technical-analysis-rsi)..."
- Без фанатизма, естественно

## 3. Что удалять

### Шаблонные блоки (удалить полностью)

```
## Заключение
[текст про накопление]
Начать торговлю можно на проверенных биржах:
- **[Bybit](ref)** — ...
- **[Bitget](ref)** — ...
- **[BingX](ref)** — ...

---

## Связанные материалы
**Основы и стратегии:** ...
**Инструменты и автоматизация:** ...
**Опыт и риски:** ...

---

Если вам интересна эта тема, загляните на страницу [О проекте](/ru/about/)
```

**Эти блоки удалять полностью.** Они:
- Содержат реферальные ссылки (навязчиво)
- Дублируются во всех статьях (шаблон)
- Не несут ценности для читателя

### Реферальные ссылки

| Тип | Оставить? |
|-----|-----------|
| В футере сайта (автоматически) | ✅ |
| 1-2 в тексте, без "register now!" | ✅ |
| 3+ в статье + "начните торговать!" | ❌ |

**Важно:** Позже нужно сделать узкие аккуратные баннеры с крутым стилем (как отдельный элемент), а не вставлять ссылки в текст.

## 4. Что добавлять

### FAQ

**Требования:**
- 3-6 вопросов
- Вопросы специфичные для темы, не шаблонные
- Ответы краткие (2-4 предложения)
- Вопросы от новичков + от продвинутых

**Пример хорошего FAQ:**
```markdown
## FAQ

**Что такое DCA простыми словами?**

DCA — это когда регулярно покупают актив на одну и ту же сумму независимо от цены. Средняя цена входа сглаживается, и не нужно угадывать момент для одной крупной покупки.

**DCA лучше разовой покупки?**

Зависит от ситуации. В сильном восходящем тренде разовая покупка может дать большую доходность. В волатильном рынке DCA даёт более комфортную цену входа.
```

### Визуалы

**Требования:**
- 1-3 изображения на статью (минимум)
- Графики, схемы, скриншоты — не стоковые фото
- Alt-текст обязателен

| Тип статьи | Визуалы |
|------------|---------|
| Индикаторы | График с индикатором |
| Стратегии | Схема логики, примеры сделок |
| Обзоры | Скриншоты интерфейса |
| Основы | Схемы, инфографика |

## 5. Требования к объёму

| Метрика | Минимум | Оптимально | Максимум |
|---------|---------|------------|----------|
| Слов | 1,500 | 1,800-2,500 | 4,000 |
| H2 разделов | 3 | 4-6 | 10 |
| Изображений | 1 | 2-4 | 8 |

**Примечание:** Качество важнее количества. Лучше 2,000 слов с примерами, чем 3,000 воды.

---

## 6. Чеклист перед публикацией

- [ ] Нет шаблонного "Заключения" с рефералками
- [ ] Нет шаблонных "Связанных материалов"
- [ ] Есть FAQ (3-6 вопросов)
- [ ] Есть "Итог" (Summary) — 1 раз
- [ ] Объём: 1,500+ слов
- [ ] 1+ изображение
- [ ] Рефералок в тексте: 0-2, без навязчивых призывов
- [ ] Проверка фактов (формулы, числа)
- [ ] Внутренние ссылки работают

DCA — эффективный способ накопить криптовалюту.

Начать торговлю можно на **[Bybit](ref)** и **[Bitget](ref)**!

## FAQ

**Сколько нужно инвестировать в DCA?**

## 8. Языки и синхронизация

- Все изменения делаются **одновременно** для всех языков (en, ru, es)
- Проверить все 3 версии статьи
- Коммит: "improve: улучшение статьи NAME (все языки)"

---

## 9. Git-процесс

1. Создать ветку `improve/article-name`
2. Внести изменения (все языки)
3. Проверить локально
4. Коммит с описанием
5. Pull request → Merge

**Пример коммита:**
```
improve: обновить dca-strategy-crypto — удалить рефералки, добавить FAQ
```
