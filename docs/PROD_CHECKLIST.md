# Чеклист перед продом и проверка публичного репо

Первый приоритет — корректность и безопасность того, что попадает в **публичный** репозиторий.

## 1. Контент и переводы

- [ ] **Заглушки:** В `src/content` нет статей с текстом «Content will be translated here» (проверка: `grep -r "Content will be translated here" src/content` → пусто).
- [ ] **Draft:** Статьи с `draft: true` не показываются на сайте — это ожидаемо, пока не снят флаг.
- [ ] **Переводы:** Для каждой статьи в RU есть соответствие в EN и ES (одинаковый slug). При добавлении новой статьи — сразу три языка или явно оставить draft в недостающих.

## 2. Перелинковка

- [ ] **Скрипт:** `node scripts/check-internal-links.mjs` → «Internal library links OK».
- [ ] **Язык ссылок:** В статьях `blog-en/` только `/en/library/...`, в `blog-es/` только `/es/library/...`, в `blog-ru/` только `/ru/library/...`.

## 3. Категории и баннеры

- [ ] **Категории:** Во frontmatter везде одна из допустимых: `crypto`, `technical-analysis`, `algo-trading`, `fundamental-analysis` (соответствует переводам в i18n).
- [ ] **Баннеры:** Если используется `bannerImage`/`heroImage` — пути к картинкам корректны, файлы лежат в `assets/` статьи или в указанном месте.

## 4. Безопасность (публичный репо — первый приоритет)

- [ ] **Секреты:** В коммитируемых файлах нет токенов, паролей, API-ключей, путей к `.github-token`. Pre-commit проверяет паттерны.
- [ ] **Пути к внутренним репо:** В отслеживаемых файлах нет путей к приватному репозиторию (локальные пути, имена папок). Исключение: общие упоминания в README/.gitignore без раскрытия структуры.
- [ ] **Документы только в private:** Планы, процедуры, аудиты с конфиденциальной информацией перечислены в `.gitignore` и не попадают в `git add` (проверка: `git status` перед коммитом).

## 5. После пуша

- [ ] **Actions:** В [GitHub Actions](https://github.com/Hampfree-hub/marketlab-academy/actions) workflow «Deploy to GitHub Pages» завершился успешно.
- [ ] **Прод:** Открыть главную, библиотеку и 2–3 статьи (RU/EN/ES), проверить отображение и ссылки.
